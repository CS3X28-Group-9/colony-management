User Manual

Basic Usage

Upon logging in, an interface similar to that shown above is presented, including a list of projects.

Clicking a project in the list presents the project page. Information about the project’s compliance with the license is present, along with a list of mice associated with the project and buttons for various actions.

An individual mouse’s page is accessible by clicking the mouse ID in the list of mice on a project’s page. More information about the individual mouse is listed there, alongside links to edit that information and to view its family tree.

In the family tree view each mouse is represented by a box headed by its strain and tube number, acting as a link to that mouse’s own family tree. The box also includes a details link directly to the mouse’s information page.

The family tree shows only mice that are direct ancestors or descendants of the requested mouse. Ancestors are shown with the father of any mouse on the left and the mother on the right.

Data Import

The data import feature is most suitable for data structured as shown below, though more columns may be provided.

Parent fields refer to the tube number and are assumed to refer to mice of the same strain as the child. Sex is expected to take values of either M or F. Blank values are assumed to be equal to the previous filled value.

Data import begins by uploading an Excel spreadsheet to the data import page linked from the front page. The project to which the mice should be added must be chosen.

The sheet name field refers to the sheet in the spreadsheet that should be used. If left blank, the most recently used sheet will be used.

The cell range must be specified to include the heading row. In a sheet structured like the example above, the relevant range would be A1:H2.

On the next page the mapping of columns must be given. Simply select the corresponding column or -- none -- if the column is not provided.

In the case of the strain, the value -- fixed -- can also be selected if the spreadsheet does not contain it in a column.

Once a mapping is chosen it can be saved and the import can be committed, finally adding the mice.

Requests

Requests can be created using the buttons on a mouse’s page or on a project’s page. If created from a project page, a mouse must then be chosen.

Additional relevant details should then be provided such as the location to which a mouse should be transferred or the reason for which they should be culled.

Once created, requests appear on the related mouse’s page and on the general Requests page accessible from the title bar.

The general requests page can be filtered using the Status and Type dropdowns. Filters selected do not apply until the Filter button is pressed.

Pending requests can be accepted or denied, and accepted requests can be marked completed if the user has permission to do so.
